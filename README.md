# 校园主干道人流与车流时空分布特征分析系统产品设计与系统方案文档

------

## 一、项目背景

随着智慧校园建设的发展，校园交通管理逐渐向数据化与智能化方向发展。校园主干道在上下课、活动等时段往往出现人车流量集中现象，如果能够对人流与车流的时空分布进行分析，将有助于了解交通高峰规律、优化交通管理策略。

近年来，基于深度学习的目标检测技术已经能够在复杂环境下稳定识别行人和车辆。其中，目标检测算法 **YOLOv8** 具有检测速度快、精度高的特点，适合实时视频分析任务。

本项目通过视频数据获取校园主干道人流与车流信息，并利用计算机视觉技术提取交通特征数据，进一步通过数据统计与可视化分析交通模式。同时引入 **大模型 API** 实现自然语言交互，使用户能够通过对话方式获取交通分析结果。

最终系统以 **Web 应用形式**呈现，用户可通过浏览器完成视频分析与交通数据交互。

------

## 六、目标检测模块

系统使用 **YOLOv8** 进行目标检测。

YOLOv8能够在单次前向传播中同时完成：

- 目标定位
- 目标分类

检测类别包括：

```
person
bicycle
car
motorcycle
```

需要注意的是：在后端分类中，将bicycle和motorcycle划分为车的范畴，即在逻辑上
人包括：people
车包括：bicycle、car、motorcycle

检测结果包括：

```
bounding box
class label（4种）
confidence score
```

------

## 七、目标跟踪模块

YOLOv8 本身只负责目标检测，不具备目标跟踪能力。因此系统引入 **DeepSort** 算法实现多目标跟踪。

DeepSort 能够为视频中的每个目标分配唯一 ID。

例如：

```
person #12
car #5
```

通过持续跟踪目标，可以得到目标运动轨迹。

------

## 八、轨迹计算与运动分析

系统为每个目标记录最近 **90帧轨迹数据**。

轨迹数据示例：

```
track_1
(120,210)
(125,212)
(130,214)
(135,216)
```

通过轨迹可以实现：

- 方向计算
- 速度计算
- 轨迹可视化

------

## 九、速度计算

系统采用 **像素速度**计算方式。

像素速度计算公式：

```
speed = √((x2-x1)^2 + (y2-y1)^2) / Δt
```

其中：

- (x1,y1) 为上一帧位置
- (x2,y2) 为当前帧位置
- Δt 为时间间隔

单位为：

```
pixel/s
```

像素速度可用于反映交通拥堵程度。

## 十、方向判断

系统通过轨迹向量判断目标移动方向。

计算方法：

首先定义屏幕上的四个基本方向对应的角度（以屏幕上方为 0°，顺时针）

```
direction_angles = {
    "North":   0°,    # 屏幕上方 (dy < 0)
    "East":   90°,    # 屏幕右方 (dx > 0)
    "South": 180°,    # 屏幕下方 (dy > 0)
    "West":  270°     # 屏幕左方 (dx < 0)
}
```

方向旋转公式

```
# raw_angle：基于屏幕坐标的原始方向角
# direction_offset：用户标定的北方方向角度（箭头指向）
# actual_angle：转换后的实际地理方向角
actual_angle = (raw_angle + direction_offset) % 360°
```

将角度映射回东南西北：

```
if 0° ≤ actual_angle < 45° or 315° ≤ actual_angle < 360°:
    return "North"  # 北
elif 45° ≤ actual_angle < 135°:
    return "East"   # 东
elif 135° ≤ actual_angle < 225°:
    return "South"  # 南
else:
    return "West"   # 西
```

最终可统计：

- 东向流量
- 西向流量
- 南向流量
- 北向流量

## 十一、ROI区域定义

ROI（Region of Interest）表示 **感兴趣区域**。

系统通过 ROI 过滤非道路区域数据。

ROI 采用 **多边形区域定义**。

示例：

```
(x1,y1)
(x2,y2)
(x3,y3)
(x4,y4)
```

只有当目标位于 ROI 内时才进行统计。

ROI判断通过 OpenCV 实现。

------

## 十二、交通计数方法

系统采用：

```
ROI过滤 + 跨线计数
```

### 跨线计数原理

在画面中设置一条 **计数线**。

当目标轨迹跨越该线时：

```
count +1
```

跨线计数可以避免重复统计。

例如：

```
------ counting line ------
```

当目标轨迹从一侧移动到另一侧时完成计数。

## 十四、数据存储设计

数据库采用 **SQLite**。

数据库包含两张表。

------

### detection_log（检测数据表）

存储每秒检测结果。

字段结构：

| 字段        | 说明     |
| ----------- | -------- |
| id          | 记录ID   |
| track_id    | 目标ID   |
| object_type | 目标类型 |
| timestamp   | 时间     |
| x           | 坐标x    |
| y           | 坐标y    |
| pixel_speed | 像素速度 |
| direction   | 移动方向 |

------

### traffic_stat（统计数据表）

每分钟生成一次统计数据。

字段：

| 字段          | 说明     |
| ------------- | -------- |
| id            | 记录ID   |
| time_slot     | 时间段   |
| person_count  | 行人数   |
| vehicle_count | 车辆数   |
| avg_speed     | 平均速度 |
| density       | 交通密度 |

------

## 十五、交通密度计算

密度计算公式：

```
density = 目标数量 / ROI面积
```

密度越高说明交通越拥挤。
