# 分类逻辑更新说明

## 📋 更新日期
2026-03-27

## 🎯 更新目标
将检测类别从原来的 `{person, car, motorcycle, bus, truck}` 更新为 `{person, bicycle, car, motorcycle}`，并在业务逻辑中将 **bicycle** 和 **motorcycle** 归为 **vehicle** 类。

---

```python

# COCO ID 映射：
# 0 - person (行人)
# 1 - bicycle (自行车)
# 2 - car (汽车)
# 3 - motorcycle (摩托车)
```

- 移除了 bus(5) 和 truck(7)

## 📊 数据流说明

### 检测流程
```
视频帧 → YOLOv8 检测 → 4 种类别 (person/bicycle/car/motorcycle)
                     ↓
                DeepSort 跟踪
                     ↓
           统计分类（新的逻辑）:
           - person: class_name == "person"
           - vehicle: class_id ∈ {1, 2, 3}
```

### 数据存储流程（方案 B）
```
检测结果 → DetectionLog 表（保存原始类别）
                    ↓
           object_type 字段值:
           - "person"
           - "bicycle"
           - "car"
           - "motorcycle"
                    ↓
        TrafficStat 表（聚合统计）:
        - person_count: 只统计 "person"
        - vehicle_count: 统计 "bicycle"+"car"+"motorcycle"
```

## 🔍 关键特性

### ✅ 保留的信息
- **原始类别名称**：数据库中仍然保存具体的类别（bicycle/car/motorcycle）
- **详细数据**：可以查询到每种具体车辆的记录

### ✅ 聚合的逻辑
- **前端显示**：只显示 person 和 vehicle 两类
- **统计数据**：vehicle_count = bicycle + car + motorcycle
- **AI 分析**：理解 vehicle 是三类车的总和

## ⚠️ 注意事项

1. **数据库需要清空**
   ```sql
   -- 建议删除所有历史数据，因为旧的分类逻辑不同
   DELETE FROM detection_log;
   DELETE FROM traffic_stat;
   ```

2. **YOLO 模型文件**
   - 确保 `yolov8n.pt` 模型支持 COCO 数据集的类别
   - 模型会自动识别 80 个类别，但代码会过滤出我们需要的 4 类

3. **前端兼容性**
   - 前端代码无需修改，因为 API 返回的字段名不变
   - 前端只需要知道 `person_count` 和 `vehicle_count`
